Getting Started

XCore is a framework that lets you manage your test cases in XML 'test scripts' instead of code. It uses JUnitDriver to execute the test. It includes features like Multi-Threading, Exception handling, block statements, Random Generators, dependency-exclusion handling, looping, plugins and utilities like File helper, Date utils and so on . This guide will help you write your first test using the XCore framework.

Our strong recommendation is to create a Java project with our Maven archetype so the project has the necessary structure and your dependencies are automagically organized. If you go this route, the first thing you need to do is to run our Maven XCore archetype from GitHub (Is this what we'll do????) and answer the questions that will be prompted by Maven. Once your project is created, the XCore Maven dependency should be automatically placed in your new project's pom file and you should have all the required libraries.

POM Snippet: 

<dependency>
  <groupId>org.finra.jtaf</groupId>
  <artifactId>jtaf-xcore</artifactId>
  <version>1.0</version>
</dependency>

Now you are ready to write some code for Hello World!

All that required is TestScript, Test Library, Test strategy and Src file.

TestScript

Add the below code in HelloWorldTest.xml under testscript folder.
<?xml version="1.0" encoding="ISO-8859-1"?>
<testsuite name="First Tests">
  <test name="HelloWorldTest" testCaseID="TC001" > <!-- testCaseID is optional -->
	  <desc>Simple Hello World Test Case</desc> <!-- desc tag and coverage are optional -->
	  <coverage>APP-001</coverage>
	  <automationValue>Regression</automationValue> <!-- a tag for filtering which tests will run in your strategy -->
	  <teststeps>
	   <ExampleCommand name="Hello world"/>
	  </teststeps>
  </test>
</testsuite>


TestStrategy

Create a file helloworld.strategy.xml under profiles/strategies

<?xml version="1.0" encoding="ISO-8859-1"?>
<execute>
  <target name="HelloWorldTest.xml" />
</execute>


Test Library

Create a file example.commands.xml under src/main/resources/testlibrary 

<?xml version="1.0" encoding="ISO-8859-1"?>

<command name="ExampleCommand" class="org.finra.jtaf.xcore.commands.ExampleCommand">
 
<usage>This command is just a simple example</usage>
 
<optionalParameters>
  <string name="text">What will get printed in the console</string>
</optionalParameters>

</command>


Java source class

Create a java class under org.finra.jtaf.xcore.commands named ExampleCommand 

package org.finra.jtaf.xcore.commands;

import org.finra.jtaf.xcore.model.exceptions.NameFormatException;
import org.finra.jtaf.xcore.model.execution.IInvocationContext;
import org.finra.jtaf.xcore.model.invocationtarget.Command;

public class ExampleCommand extends Command {

  public ExampleCommand(String name) throws NameFormatException {
    super(name);
  }

  protected void execute(IInvocationContext ctx) throws Throwable {
    String name=getStringorDefault(text,"Hello World");
    System.out.println(name);
  }
}

Run Configuration
Check jtaf.properties under profiles folder

client=profiles/clients/firefox.properties
strategy=profiles/strategies/helloworld.strategy.xml

Now if you run your test, you will see a Firefox browser start up and running your test scenario. You can also run against other browsers which are supported by WebDriver. Please refer to Client Properties page for other useful configurations.
