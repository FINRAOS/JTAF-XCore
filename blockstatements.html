Block statements

It is a collection of commands that can be used as a parameter to another command or on their own. Each block has its own context.

2 types of block statements are

1. Expect Failure
2. Ignore Errors


Example for Expect Failure

A block that is used when there is supposed to be an exception thrown by a command. If an exception is thrown, the test will stop executing commands within the block but attempt to execute the remainder of the script. If an exception is not encountered, an ExpectFailureException is thrown instead. If the ExpectFailure is nested within another ExpectFailure, this exception will still be thrown (doesnâ€™t count as an expected failure). 

<test name="Nested Block Testing">
	 <automationValue>High</automationValue>
  <teststeps>
	<block>
	  <block>
		<expectFailure>
			<FailHere/>
		</expectFailure>
	  </block>
	  <string name="name">huh</string>
	  <RandomGenerator method="string"  string="name"/>
	</block>
  </teststeps>
</test>

Example for Ignore Error

Surrounds a block of commands and if an error is encountered, it will stop execution of that block but the rest of the script will continue. The exception is then thrown when the test is done executing.
<test name="Errors">
  <automationValue>High</automationValue>
  <teststeps>
    <ignoreErrors>
	  <FailHere/>
    </ignoreErrors>
  </teststeps>
</test>
