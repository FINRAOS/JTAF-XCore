Contribute

We encourage everyone to make contribution to the project! If you want to make XCore better with new features, enhancements or bug fixes, here's how you can contribute.

  • Create a GitHub account
  • Submit a ticket in GitHub project (please verify the ticket doesn't already exist by searching here)
  • We will assign you to the ticket once the ticket is approved.
  • Fork the repository in GitHub
  • Code code code!
  • Create unit/integration tests
    • For every class you are writing/modifying, you must create/modify unit or integration tests
    • You should not break existing tests
  • Build
    • mvn package
      • Compile code
      • Create package jar
    • mvn verify
      • Compile
      • Run integration tests (unit tests)
  • Make a pull request
    • Include reference to the ticket
    • After review, the pull request will be merged to the master

When you make a pull request, you must sign off your work. You can automatically sign off by using "git commit -s" command. When you sign off, you are making an agreement to the following:

Report Bugs

When you encounter bugs, please try to find out if the issue has already been reported by searching here. Once you identify that your issue is new, please create a new ticket by clicking 'New Issue'. Please make sure you provide a clear description of the issue along with a testable scenario.

If you want to get involved in fixing bugs, first read the steps outlined above under "Contribute". Then, you can notify the team by adding a comment. Then we will assign the bug to you and you can start developing your code. Once you complete your code, build and test it, please make a pull request for review. Once the review is complete, your code fix will be merged into master branch within the JTAF-XCore repository.

Communicate
